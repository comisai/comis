Comis agents have access to 50+ tools spanning file operations, web access, messaging, scheduling, media processing, browser automation, context management, and administration. This page is the master reference -- every tool listed with its category, description, and link to detailed documentation.

When an agent receives a message, it decides which tools to use based on what you asked it to do. Need a daily briefing? The agent uses the `cron` tool. Want to send a message to another channel? It uses `message`. Need to analyze an image someone sent? It reaches for `image_analyze`.

The tools listed below are everything your agents have at their disposal. Each tool is linked to its detailed guide page where you will find parameters, examples, and tips for effective use.

## Understanding Tool Categories

Tools fall into three tiers:

| Tier | Count | Description |
|------|-------|-------------|
| **Built-in tools** | 12 | Core capabilities from the coding agent -- file operations, shell commands, web search, and browser automation. Work out of the box. See [Built-in Tools](/skills/built-in-tools). |
| **Platform tools** | 30+ | Comis-specific capabilities -- messaging, scheduling, sessions, media, infrastructure, and moderation. Documented on this tab. |
| **Custom tools** | Unlimited | Your own skills and MCP server tools. You create these to give agents new abilities. See [Skills](/skills/index). |

Tool access is controlled by [tool policy](/skills/tool-policy). By default, agents have access to all tools, but you can restrict access per agent using profiles (like `coding` or `messaging`) or fine-grained allow/deny lists.

<Info>
All tool names use `snake_case` -- for example, `web_search`, `sessions_spawn`, and `memory_store`. When referencing tools in configuration files (such as tool policy allow/deny lists), use these exact names.
</Info>

### Built-in Tools

Built-in tools come from the coding agent and handle file operations, shell commands, web access, and browser automation. They are always available unless explicitly denied by tool policy. These tools can be individually toggled in your `config.yaml` under the `builtinTools` section.

| Tool | Description | Guide |
|------|-------------|-------|
| `read` | Read file contents with line numbers and pagination | [Built-in Tools](/skills/built-in-tools) |
| `write` | Create or overwrite files, auto-creates parent directories | [Built-in Tools](/skills/built-in-tools) |
| `edit` | Search-and-replace edits on existing files | [Built-in Tools](/skills/built-in-tools) |
| `notebook_edit` | Cell-level Jupyter notebook editing | [Built-in Tools](/skills/built-in-tools) |
| `grep` | Search file contents using regex patterns | [Built-in Tools](/skills/built-in-tools) |
| `find` | Find files by glob pattern | [Built-in Tools](/skills/built-in-tools) |
| `ls` | List directory contents | [Built-in Tools](/skills/built-in-tools) |
| `exec` | Run shell commands with timeout and safety checks | [Built-in Tools](/skills/built-in-tools) |
| `process` | Manage background processes (list, kill, status, log) | [Built-in Tools](/skills/built-in-tools) |
| `web_search` | Search the web using 8 providers with automatic fallback | [Web Tools](/agent-tools/web-tools) |
| `web_fetch` | Fetch web page content with readable text extraction | [Web Tools](/agent-tools/web-tools) |
| `browser` | Headless browser automation (16 actions) | [Browser](/agent-tools/browser) |

### Scheduling

Scheduling tools let agents create automated tasks that run on a timer. Agents can set up recurring cron jobs, one-time reminders, and manage their own heartbeat -- a periodic wake-up that triggers routine maintenance or check-in tasks.

| Tool | Description | Guide |
|------|-------------|-------|
| `cron` | Create and manage scheduled jobs (8 actions, 3 schedule types) | [Scheduling](/agent-tools/scheduling) |
| `heartbeat_manage` | Configure agent heartbeat timing and triggers | [Scheduling](/agent-tools/scheduling) |

### Messaging

The message tool works across all connected channels -- Discord, Telegram, Slack, WhatsApp, and every other supported platform. Agents can send new messages, reply to specific messages, add emoji reactions, edit or delete messages, fetch message history, and send file attachments.

Agents can also send messages to channels they are not currently conversing in, making the message tool ideal for notifications, alerts, and cross-channel coordination.

| Tool | Description | Guide |
|------|-------------|-------|
| `message` | Send, reply, react, edit, delete, fetch, and attach messages | [Messaging](/agent-tools/messaging) |
| `notify_user` | Send proactive notifications, alerts, and reminders to the user | [Messaging](/agent-tools/messaging) |

### Sessions

Session tools let agents manage their conversations, communicate with other agents, spawn sub-agents for specialized tasks, and run multi-step pipelines. These are the tools that enable multi-agent coordination.

For example, a lead agent can spawn a researcher sub-agent and a writer sub-agent to work in parallel, then collect their results using the pipeline tool.

| Tool | Description | Guide |
|------|-------------|-------|
| `session_tool` | Unified session management (search, status, history, list) | [Sessions](/agent-tools/sessions) |
| `session_status` | View current session info (agent, channel, message count) | [Sessions](/agent-tools/sessions) |
| `sessions_list` | List active sessions with optional filters | [Sessions](/agent-tools/sessions) |
| `sessions_history` | Browse conversation history for any session | [Sessions](/agent-tools/sessions) |
| `sessions_send` | Send a message into another session (3 modes) | [Sessions](/agent-tools/sessions) |
| `sessions_spawn` | Spawn sub-agent sessions with workspace isolation | [Sessions](/agent-tools/sessions) |
| `subagents` | Manage running sub-agents (list, kill, steer) | [Sessions](/agent-tools/sessions) |
| `agents_list` | List all configured agent IDs | [Sessions](/agent-tools/sessions) |
| `pipeline` | Execute multi-step DAG workflows (9 actions) | [Sessions](/agent-tools/sessions) |

### Media

Media tools process images, audio, video, and documents. Agents can analyze images with vision AI, convert text to speech, transcribe voice messages, describe video content, and extract text from documents like PDFs. These tools require appropriate AI providers to be configured (for example, vision capabilities for image analysis, and STT providers for transcription).

| Tool | Description | Guide |
|------|-------------|-------|
| `image_analyze` | Vision AI image analysis (file, URL, base64, attachment) | [Media](/agent-tools/media) |
| `tts_synthesize` | Text-to-speech audio generation | [Media](/agent-tools/media) |
| `transcribe_audio` | Audio and voice transcription to text | [Media](/agent-tools/media) |
| `describe_video` | Video description using vision AI | [Media](/agent-tools/media) |
| `extract_document` | Document text extraction (PDF, CSV, TXT, and more) | [Media](/agent-tools/media) |
| `image_generate` | Generate images from text prompts, delivered to the current channel | [Media](/agent-tools/media) |

### Memory

Memory tools give agents access to their long-term memory. Agents automatically build up memory over time, but these tools let them explicitly search, read, and store information. Memory uses hybrid text and vector search for semantic matching, so agents can find relevant memories even with different wording.

| Tool | Description | Guide |
|------|-------------|-------|
| `memory_tool` | Unified memory management (get, search, store, manage) | [Sessions](/agent-tools/sessions) |
| `memory_search` | Semantic memory search using text and vector similarity | [Sessions](/agent-tools/sessions) |
| `memory_get` | Read specific memory entries by path | [Sessions](/agent-tools/sessions) |
| `memory_store` | Store a memory entry (with built-in secret detection) | [Sessions](/agent-tools/sessions) |

### Context

Context tools give agents access to their conversation DAG (directed acyclic graph), enabling deep search and recall across compressed conversation history. The context tool consolidates search, recall, inspect, and expand operations into a single interface.

| Tool | Description | Guide |
|------|-------------|-------|
| `context_tool` | Unified context DAG management (search, recall, inspect, expand) | [Sessions](/agent-tools/sessions) |

### Infrastructure

The gateway tool gives agents control over the Comis configuration and gateway. It supports reading, patching, and applying configuration changes, as well as restarting the gateway and managing environment variables. The background tasks tool lets agents manage long-running operations.

| Tool | Description | Guide |
|------|-------------|-------|
| `gateway` | Configuration management and gateway control (10 actions) | [Infrastructure](/agent-tools/infrastructure) |
| `background_tasks` | Manage background tasks (status, read output, cancel) | [Infrastructure](/agent-tools/infrastructure) |

### Supervisor

Supervisor tools require admin-level trust. They manage the entire platform, not just the current agent's session. Only agents with the `supervisor` profile or explicit `allow` entries can use these tools.

<Warning>
Supervisor tools can modify agents, delete sessions, flush memory, and manage gateway tokens. Grant these tools only to trusted agents that need fleet-wide control. See [tool policy](/skills/tool-policy) for how to restrict access.
</Warning>

| Tool | Description | Guide |
|------|-------------|-------|
| `agents_manage` | Agent fleet management (create, update, suspend, resume) | [Infrastructure](/agent-tools/infrastructure) |
| `obs_query` | Observability and metrics queries (diagnostics, billing) | [Infrastructure](/agent-tools/infrastructure) |
| `sessions_manage` | Session lifecycle management (delete, reset, export) | [Infrastructure](/agent-tools/infrastructure) |
| `memory_manage` | Memory lifecycle management (stats, browse, flush) | [Infrastructure](/agent-tools/infrastructure) |
| `channels_manage` | Channel adapter management (enable, disable, restart) | [Infrastructure](/agent-tools/infrastructure) |
| `tokens_manage` | Gateway token management (create, revoke, rotate) | [Infrastructure](/agent-tools/infrastructure) |
| `models_manage` | Model catalog queries (list, test) | [Infrastructure](/agent-tools/infrastructure) |
| `skills_manage` | Skill lifecycle management (list, import, delete) with per-agent scope | [Infrastructure](/agent-tools/infrastructure) |
| `mcp_manage` | MCP server management (connect, disconnect, reconnect) | [Infrastructure](/agent-tools/infrastructure) |

### Platform Actions

Platform action tools perform moderation and administration tasks specific to each messaging platform. Each tool wraps the native API of its platform, letting agents kick users, manage roles, pin messages, create channels, and more -- all from within a conversation.

The available actions vary by platform. Discord has the most actions (19), followed by Telegram (12) and Slack (12), then WhatsApp (11). See the [Platform Actions](/agent-tools/platform-actions) guide for the full list of actions per platform.

| Tool | Actions | Guide |
|------|---------|-------|
| `discord_action` | 19 actions (kick, ban, roles, threads, channels, and more) | [Platform Actions](/agent-tools/platform-actions) |
| `telegram_action` | 12 actions (pin, poll, sticker, ban, promote, and more) | [Platform Actions](/agent-tools/platform-actions) |
| `slack_action` | 12 actions (pin, topic, archive, channels, invites, and more) | [Platform Actions](/agent-tools/platform-actions) |
| `whatsapp_action` | 11 actions (group management, participants, settings, and more) | [Platform Actions](/agent-tools/platform-actions) |
